## Notes for next session

**Hard-won finding to preserve.** The Claude Preview MCP's
`preview_start` does **NOT** honor `runtimeExecutable` /
`runtimeArgs` from `.claude/launch.json`. It always runs
`npm run dev` regardless of which launch.json config name you
request. On TheLaunchPad this loads the parent worktree's prod
`.env.local` under Next.js's workspace-root inference (Next.js
detected `C:\Users\14252\package-lock.json` as the workspace root
and picked up env files from the parent dir). The first smoke test
attempt during Session A.5 connected to **prod** Supabase as a
result. The failed login was a 400 (no actual prod data modified),
but the next attempt with a real password would have. Workaround
that works: direct Bash `run_in_background` with the dotenv-cli
wrapper on a different port. Documented in
`supabase/scratch-replay-notes.md` and the in-repo handoff so
Mac-Claude doesn't get burned.

**Stray commit on the feature branch.** `git log main..HEAD` shows
five commits, four mine and one not: `3d5c222 vault: build 66d
completion + 66 close-out`. That commit predates this session and
came from another worktree (the `cranky-curie-9e4933` worktree on
`claude/cranky-curie-9e4933`). When `build-65b-session-a` eventually
merges to main, `3d5c222` goes with it. Probably fine — it's a vault
completion note that should be on main anyway — but worth a quick
`git show 3d5c222` review before the merge in case it has unintended
effects.

**Build 42 deviation rationale.** Eric's prod `user_id`
(`7c55cdd0-2cbf-4c8a-8fdd-e141973ade94`) cannot exist in scratch's
`auth.users` because we deliberately did not SQL-seed an auth.users
row (per the build52 lesson: GoTrue panics on NULL token columns when
SQL-seeded). The `user_organizations` membership row is re-created
in `seed-scratch.sql` for the dashboard-created test user
(`bc93b88d-...`), so scratch ends in the same shape as prod — one
admin row in `user_organizations` for AAA — just attached to a
different person. The deviation is reproducible:
`scratch-replay-notes.md` "Reproducing this replay" section walks
through the steps for any future fresh scratch project.

**Tooling assumption.** `dotenv-cli ^11.0.0` was added to devDeps
without asking — the prompt was specific that it should land. The
optional `dev:scratch` script in `package.json` was NOT added (prompt
said to ask first). Future sessions can add the script if it's noisy
enough to bother with.

**`mock-photo-tags.ts` is in `src/` but unused.** Type-checker still
validates it. Future cleanup could either delete it or move it under
