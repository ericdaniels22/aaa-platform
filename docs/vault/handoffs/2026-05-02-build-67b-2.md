### Plan-spec corrections re-applied uniformly across Tasks 16–25

Same two corrections from prior session, applied to every new route:
1. `requirePermission` returns discriminated union `{ok: true; userId} | {ok: false; response}`. Use `if (!auth.ok) return auth.response;` not the plan's `if (denied) return denied;`.
2. `getActiveOrganizationId` returns `Promise<string | null>`. When orgId flows into a strict-string param of a lib function (createInvoice, createTemplate, etc.) or a NOT NULL insert column, guard with `if (!orgId) return NextResponse.json({ error: "no active org" }, { status: 400 });`.
