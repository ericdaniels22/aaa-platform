## Notes for next session

- The four files in `.claude/skills/*/SKILL.md` and `.claude/commands/*.md` are verbatim copies of the Build 66c spec section. Don't tweak the runbooks without first re-reading the spec to understand why each rule is there — particularly the "Non-negotiable rules" sections, which exist because each rule maps to a failure mode (invented progress, fabricated decisions, dumping full file contents instead of distilling, etc.).
- **Gitignore gotcha worth remembering.** Git's docs (`gitignore(5)`): "It is not possible to re-include a file if a parent directory of that file is excluded, since git doesn't list excluded directories." So `.claude/` + `!.claude/skills/` doesn't work — you have to use `.claude/*` (wildcard children) so each entry is evaluated against the negation list. The `.gitignore` already uses the same pattern for `.yarn/*` (lines 7–11) and `/out/*` (lines 18–20).
- The skill's step 6 specifies a single commit (`vault: handoff for build {id} on {date}`). The user's two-commit pattern overrode this only for the self-test, because commit 1 was the skills themselves — a logically separate change from the meta-handoff. Future invocations of `/handoff` should follow the SKILL.md as written: one commit, just the vault changes.
