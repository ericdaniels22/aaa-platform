## Notes for next session

**The auto-save concurrency model is now load-bearing.** With server-side `updated_at_snapshot` enforced on the three child PUT routes, two clients editing the same estimate will produce 409s on the slower writer. The hook handles 409 by setting `staleConflictRef` and short-circuiting all future saves, with an indefinite toast directing the user to refresh. This is correct UX for now, but if 67b ever ships features that ALSO mutate `estimates.updated_at` (e.g. invoice creation triggering an estimate status flip to `converted`), each of those routes needs to either (a) bump `updated_at` so any open editor knows to refresh, or (b) skip the bump so existing editors stay valid. Lean (a). Document the rule in the build 67b plan.

**The retry pattern in `use-auto-save.ts` deliberately does NOT retry the original payload for line-item field saves.** Per-item retry calls `performLineItemSave(item)` again with the captured `item` — but if the user has typed since, the next debounce cycle's `performLineItemSave` wins (with the latest state) and the retry is a no-op due to the in-flight guard. This is "eventually consistent" rather than "exactly the user's last keystroke," which is the right tradeoff for typing UX. Reorder retries DO reuse the captured payload because `handleDragEnd` is an explicit user action, not continuous input.

**`lib/postgrest.ts` is the canonical place for any future PostgREST-related escape helpers.** If more callers surface (e.g. `.and(...)` with user input, or `referencedTable` filters), add to that module rather than scattering inline escapes.

**`lib/api-errors.ts` is similarly canonical for the redactor.** When the broader sweep happens, the apply is mechanical: import `apiDbError`, replace `NextResponse.json({ error: error.message }, { status: 500 })` with `apiDbError(error.message, "<METHOD> <route> <op>")`.

**The status badge map dedup also pulls double duty for 67b.** When `EstimatesInvoicesSection` grows an Invoices half (currently a stub), the same status logic likely applies. `lib/estimate-status.ts` is the obvious extension point — add `INVOICE_STATUS_BADGE_CLASSES` etc., or rename the file to `status-badges.ts` if the line between the two domains blurs.
