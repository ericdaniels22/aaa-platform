# Build 14j Handoff — 2026-04-30

## What shipped this session

- **[[build-14j]] — Intake Form Builder UX overhaul.** Replaces the 483-line modal-driven `/settings/intake-form` page with a three-column WYSIWYG editor: left palette (field types + presets), center canvas (live form rendering, click-to-select, drag-and-drop, mobile-width toggle, Edit/Test mode), right inspector drawer. Auto-save (debounced 1500ms) replaces the manual Save button. Plan: `docs/superpowers/plans/2026-04-30-build-14j-form-builder-ux.md`.
- **Schema migration:** [supabase/migration-build14j-prep-form-config-versioning.sql](../../../supabase/migration-build14j-prep-form-config-versioning.sql) drops the singleton `form_config_org_key` (one row per org) added by build46 and replaces it with a composite `form_config_org_version_key (organization_id, version)`. Restores the original 14f append-only history design that 14j depends on. **Side effect:** also fixes a pre-existing latent bug in the unchanged `/api/settings/intake-form` POST route — the `INSERT` with `version+1` had been silently 500'ing on every save after the first since build46 landed. Applied to prod (`rzzprgidqbnqcdupmpfe`) via Supabase MCP `apply_migration` after a `(organization_id, version)` duplicate-detection pre-flight returned zero rows.
- **Two new API routes:** `GET /api/settings/intake-form/versions` (last 20 versions, org-scoped) and `POST /api/settings/intake-form/restore` (copies a target version's config forward as a new row, preserves history). Both filter by `getActiveOrganizationId(supabase)` to match the existing route's tenant scoping.
- **`maps_to`-based submit routing.** Refactored [src/components/intake-form.tsx](../../../src/components/intake-form.tsx) `handleSubmit` to read values via `valueByMapsTo("contact.first_name")` etc. instead of literal `getVal("first_name")`. Inspector exposes a "Maps to" dropdown with optgrouped destinations (Contact / Job / Adjuster). AAA-compat preserved via `getVal` fallback for the legacy unmapped adjuster fields.
- **Per-option color picker for pill fields.** Adds `bg_color`/`text_color` to `FormFieldOption` (CSS values, applied via inline style — Tailwind JIT can't extract dynamic class strings). 8 swatches in the inspector; both the canvas FieldPreview and the public `IntakeForm` pill renderer honor the colors.
- **Test mode UX.** `TestMode` component reuses the public `IntakeForm` with a new `testMode={true}` prop that short-circuits the Supabase writes after per-config validation but before the hardcoded canonical check. `viewMode` lifted from Canvas to the page so the palette, Add section, and Inspector chrome all collapse in Test mode.
- **Misc UX:** "Saved · v{N}" pill simplified to just "Saved" (history dropdown still shows version numbers). "Add section" moved from the page header to below the last section in the canvas, with an inline title input that swaps in-place.
- **TestCo seeded with AAA's intake form** as TestCo `form_config` v103 (direct INSERT-from-SELECT) so Eric can continue exercising real intake submissions during ongoing testing.
- **PR #44 merged via rebase** to keep individual task commits on main. Final main HEAD before handoff: `a651057 fix(14j): route intake submit by maps_to (was: literal field IDs)`. Production Vercel deploy succeeded at `aaaplatform-pxgj2foyz-aaa-disaster-recovery-e5661f28.vercel.app` (aliased to `aaaplatform.vercel.app`).

## Notes for next session

- **The schema fix is also a fix for a pre-existing latent bug.** The unchanged `/api/settings/intake-form` POST route had been silently 500'ing on every save after the first save per org since build46 (because the singleton unique constraint on `organization_id` blocked further inserts). Eric had been editing AAA's form for months without noticing — the prior toast was a generic "Save failed" that nobody dug into. **Worth scanning other settings pages for similar latent bugs:** any POST route that does `INSERT` with a `version+1` pattern against a table whose unique index is `(organization_id)` instead of `(organization_id, version)` will have the same issue.
- **TestCo's pre-AAA-shape form_config versions are still in history** (v1–v102 with `custom_<timestamp>` field IDs and no `maps_to`). Restoring those via the version pill brings back the broken state — submits from those versions hit the hardcoded canonical check and fail. Don't restore them.
- **The `MAPS_TO_GROUPS` constant in [src/components/form-builder/inspector.tsx](../../../src/components/form-builder/inspector.tsx)** hardcodes the canonical destinations (`contact.first_name`, `job.damage_type`, etc.). If more columns get added to `contacts`, `jobs`, or the adjuster flow later, this list needs an update — the dropdown is the only place users can select a `maps_to`.
- **AAA's seed has `is_default: true` on the canonical fields, with `maps_to` populated.** TestCo's v103 inherited that flag — meaning the Inspector locks the type for those fields. This is a feature, not a bug: it prevents accidental type changes on canonical fields. To experiment with a different type on a TestCo "First Name" field, either delete + re-add as a custom, or temporarily flip `is_default` via SQL.
- **AAA's `urgency` pill options use legacy Tailwind class strings** (e.g. `color: "bg-[#FCEBEB] text-[#791F1F] border-[#791F1F]/30"`). The new pill renderer prefers `bg_color`/`text_color` (inline style) when set, falls back to `color` (class string) when only that's present. AAA-style urgency pills render the same way they did pre-14j. Migrating to the new color system is one inspector click per option.
- **Workspace-root inference warning still appears** in dev logs ("Next.js inferred your workspace root, but it may not be correct" — detects parent worktree's `package-lock.json`). Cosmetic only — `.env.local` loaded correctly from the worktree root. Could be silenced by setting `outputFileTracingRoot` in `next.config`. Skipped this session as out-of-scope.
- **Four iterations of user-driven UX feedback after the initial 14j ship** uncovered issues the plan didn't anticipate: (1) test mode chrome was distracting → palette + Add section + Inspector hidden in Test mode; (2) version counter on "Saved" pill was noise → dropped, dropdown still numbered; (3) pill colors needed user-pickable swatches → 8 presets per option via inline style; (4) Add section button at top of page was disconnected from where the section lands → moved below last section with inline title input. All four shipped + verified end-to-end in the same session.
- **A real test job and contact ("Eric Test TestCo", `WTR-2026-0001`) were created in TestCo during verification.** Real data, kept for Eric to inspect. Delete via the Jobs UI when no longer needed.
- **Hydration warning fix worth knowing about:** the version pill outer wrapper is a `<div role="button" tabIndex={0}>` rather than a `<button>` because the Retry sub-button (rendered when status is "error") is a real button — nesting was emitting React's "<button> cannot be a descendant of <button>" warning during hydration.
